title: Performance Is a Feature
date: 28 May 2026
tag: Engineering
excerpt: Sixty frames per second is not a luxury. It is a sign of respect for the user.
---

Performance is often treated as a technical concern, but it is a core part of the user experience. Every delay, stutter, and unnecessary wait tests a person’s patience.

Sixty frames per second is not an extra layer of polish. It is the baseline set by modern operating systems and browsers. Falling below that baseline makes an interface feel heavy.

Performance is not only about raw speed. It is also about response. When someone presses a button, the interface should acknowledge it immediately through motion, sound, or a visible change.

A feature cannot feel well designed if it performs poorly. Response time, animation length, and transition timing are all design decisions.

People may not remember every animation, but they remember how a product made them wait. Speed is a form of respect.
